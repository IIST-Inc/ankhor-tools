Application Guide

What Can I Do With Ankhor Key Plus?

Choose your use case. Each scenario shows the Ankhor function area, suggested commands, integration idea, and where IIST can provide implementation support.

Overview

Ankhor Key Plus provides a fixed set of hardware-rooted application security functions. It is not an unlimited custom firmware product by default. Users can use the provided CLI/API functions directly, or request IIST integration support for production deployment.

Need this implemented in your product?

Contact IIST for integration support, provisioning scripts, user-specific demos, or production workflow design.

01 I want users to log in without passwords
Function area
Standard FIDO2 / WebAuthn authentication.
Important
This scenario only explains the use case. For browser-level registration and authentication trials, use the external WebAuthn.io link on the homepage.
Problem
Users still rely on passwords that can be reused, phished, leaked, or stolen.
Typical deployment
Web app login, VPN login, admin dashboard login, enterprise identity provider login.
Use these functions
Register the Ankhor key as a compatible authenticator in the user authentication platform.
Suggested integration
Register the Ankhor key as a FIDO-compatible authenticator in the user's existing authentication platform.
User work
Use an existing FIDO/WebAuthn-compatible platform.
IIST optional service
IIST can support user demo setup and authentication workflow discussion.
02 I want to protect database passwords on a gateway or server
Function area
PM Credential Storage.
Problem
Database passwords are often stored in config files, environment variables, startup scripts, or source code.
Use these functions
pm add -t "database" -n "main-db" --password "<DB_PASSWORD>"
pm get -t "database" -n "main-db"
pm del -t "database" -n "main-db"
Suggested integration
Application retrieves the database password from Ankhor during startup instead of reading plaintext config.
User work
Modify application startup or backend code to request the credential from Ankhor.
IIST optional service
Linux/OpenWrt startup script integration, Python/C examples, provisioning scripts.

Function area
AKM API Key Manager.
Problem

Typical deployment

04 I want hardware-protected OTP/TOTP codes
Function area

Problem

otp del -t "vpn" -n "admin"

Suggested integration
